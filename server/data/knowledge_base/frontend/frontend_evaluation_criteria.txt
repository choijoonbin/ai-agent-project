프론트엔드 개발자 평가 기준 (총점 100점)
1. Web Front-End 기술 역량 (35점)
* HTML/CSS/JavaScript 기본기 및 웹 표준/웹 접근성 이해
* Vue/React/Angular 등 프레임워크 활용 능력
* 상태 관리, 라우팅, 컴포넌트 아키텍처 설계 능력
* 브라우저 렌더링, DOM, 이벤트 처리 구조 이해
* REST API, GraphQL, WebSocket 등 연동 경험
2. UI/UX 구현 능력 (20점)
* 사용자 경험을 고려한 화면 구성 및 인터랙션 설계
* 반응형 / 적응형 UI 구현 능력
* 디자인 시스템 또는 UI 컴포넌트 라이브러리 활용
* 제품 특성과 비즈니스 목적에 맞는 UX 개선 활동 경험
3. 성능 최적화 및 품질 관리 (20점)
* 렌더링 성능, 번들 최적화, Lazy Loading, 코드 스플릿 등
* Lighthouse / Chrome DevTools 등 성능 측정 도구 사용 능력
* 장애/이슈 해결 능력 및 디버깅 능력
* 크로스 브라우저 대응, 테스트 자동화 경험
4. 협업 및 커뮤니케이션 (15점)
* 기획/디자인/백엔드 등 타 직군과의 협업 능력
* 기술적 결정을 근거 기반으로 설득하고 공유하는 능력
* 코드리뷰 문화 및 협업 도구(Jira, Confluence, Git 등) 활용 능력
5. 문서화 및 코드 품질 (10점)
* 코드 일관성, 유지보수성 및 가독성
* 테스트 코드 작성(Jest, Cypress, Vitest 등)
* 기술 문서, 개발 가이드를 작성하고 공유한 경험

**다른 직군 경험과의 차이점 (평가 시 주의사항)**
* Backend 개발 경험을 Frontend 역량으로 평가할 때:
  - 서버 로직 개발 ≠ UI/UX 구현: Backend는 API 개발, 데이터베이스 설계를 의미하지만, Frontend는 사용자 인터페이스 구현, 사용자 경험 설계를 의미합니다.
  - 서버 성능 최적화 ≠ 브라우저 렌더링 최적화: Backend는 서버 처리 성능, 데이터베이스 쿼리 최적화를 의미하지만, Frontend는 렌더링 성능, 번들 크기 최적화, 코드 스플릿을 의미합니다.
  - API 설계 경험 ≠ 프론트엔드 상태 관리: Backend는 REST API, GraphQL 설계를 의미하지만, Frontend는 상태 관리, 컴포넌트 아키텍처, 라우팅을 의미합니다.
* 디자이너 경험을 Frontend 역량으로 평가할 때:
  - 디자인 툴 활용 ≠ 프론트엔드 개발: 디자이너는 Figma, Sketch 등 디자인 툴을 사용하지만, Frontend는 HTML/CSS/JavaScript로 실제 구현을 의미합니다.
* 평가 시 다른 직군 경험이 Frontend 역량의 일부 요소에만 해당할 수 있음을 고려하고, 보수적으로 평가하세요.
