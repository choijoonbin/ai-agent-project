QA / Test Engineer Best Practices
1. Test Strategy
* 요구사항/리스크 기반 테스트 전략 수립
* 기능/비기능(성능, 보안, 사용성) 테스트 균형 확보
* 테스트 범위와 깊이를 명확히 정의
* 테스트 환경/데이터 전략 사전 준비
2. Test Design
* 명확한 입력/출력/사전조건/사후조건 정의
* 엣지 케이스/에러 케이스 포함
* 테스트 케이스 재사용/템플릿화
* 추적성(요구사항 ↔ 테스트 케이스 ↔ 버그) 확보
3. Test Automation
* 자동화 우선순위 기준(빈도/위험도/반복도) 정의
* UI/API/단위 테스트 레벨별 역할 분리
* CI 파이프라인에 자동화 테스트 통합
* 자동화 스크립트 유지보수성 고려(페이지 오브젝트 등)
4. Defect Management
* 재현 가능한 버그 리포트(환경, 재현 절차, 기대/실제 결과)
* Severity/Priority 기준 일관성 유지
* 버그 라이프사이클(등록–분석–수정–검증–종결) 관리
* RCA(근본 원인 분석)와 재발 방지 액션 정리
5. Quality Advocacy
* 품질 이슈를 데이터와 사례로 설명
* 개발/기획과 협업하여 테스트 가능성을 설계 단계에서 반영
* 릴리즈 Go/No-Go 판단 근거 제시
