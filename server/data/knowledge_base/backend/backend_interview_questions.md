백엔드 개발자 면접 평가 질문 예시 (별도 모음)
1. 기술 역량 및 아키텍처 질문

REST API와 GraphQL의 차이점과 각각 적합한 상황을 설명해주세요.

트랜잭션 격리 수준(Transaction Isolation Level)의 종류와 차이를 설명해주세요.

MSA를 적용한 경험이 있다면 서비스 간 통신, 장애 대응 전략, 공통 모듈 관리 방식에 대해 말해주세요.

데이터베이스 인덱스가 어떻게 동작하는지 설명해주세요.

동시성과 병렬 처리 관련 문제를 해결한 경험을 설명해주세요.

2. 시스템 설계 및 확장성 질문

대량 트래픽 또는 성능 저하 문제가 발생했을 때 어떤 방식으로 문제를 해결했는지 구체적인 사례를 설명해주세요.

캐시 전략(Redis, CDN 등)을 사용한 경험과 도입 효과를 설명해주세요.

파일 업로드, 배치 처리, 실시간 알림(웹소켓 등) 중 구현 경험과 고려한 요소를 설명해주세요.

장애 모니터링을 위해 어떤 도구와 지표를 사용했습니까? (예: APM, 로그, 메트릭)

3. 문제 해결 능력 질문

실제 서비스 장애를 해결한 경험이 있다면 근본 원인 분석과 해결 과정, 재발 방지 조치를 설명해주세요.

다른 개발자가 남긴 복잡한 레거시 코드를 개선한 경험이 있나요?

성능 최적화를 위해 DB 구조 또는 쿼리를 개선한 사례를 설명해주세요.

4. 협업 및 커뮤니케이션 질문

프론트엔드 또는 모바일 팀과 API 계약(스펙)을 조율한 경험을 말해주세요.

기술적 의견 충돌이 있었을 때 어떤 기준으로 의사결정했습니까?

코드 리뷰 과정에서 주고받은 의미 있는 경험을 설명해주세요.

5. 테스트 및 코드 품질 관리 질문

테스트 코드를 작성하는 기준과 테스트 전략(Unit/Integration/E2E)을 설명해주세요.

CI/CD 파이프라인을 구축하거나 개선한 경험이 있습니까?

리팩터링 과정에서 어떤 기준을 적용했는지 사례를 설명해주세요.
