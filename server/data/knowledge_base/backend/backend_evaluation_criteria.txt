백엔드 개발자 평가 기준 (총점 100점)
1. 기술 역량 및 아키텍처 이해도 (35점)
* Java/Spring, Node.js, Python 등 주요 언어 및 프레임워크 이해도
* 서버 구조, REST API, GraphQL, gRPC 등 설계 및 개발 역량
* 데이터베이스 설계/쿼리 튜닝 능력 (RDBMS, NoSQL)
* 시스템 아키텍처 구성(모놀리식, MSA, 이벤트 기반 등) 이해
* 인증/보안(JWT, OAuth2, 암호화) 및 API 버전 관리
2. 시스템 설계 및 확장성 고려 (25점)
* 고가용성, 확장성, 안정성 설계 능력
* 대용량 트래픽 처리, 캐싱, 메시지큐 활용 경험
* ERD 설계, DB 모델링 및 트랜잭션 처리
* 장애 대응 및 운영 관점 문제 해결 역량
3. 문제 해결 능력 (20점)
* 장애 분석 및 원인 파악 능력
* 복잡한 요구사항을 구조화하고 해결하는 능력
* 로그 분석, 진단 도구 활용 경험
* 비기능 요구(성능, 보안, 비용) 고려 능력
4. 협업 및 커뮤니케이션 (10점)
* 프론트/기획/디자인/DevOps 등 타 직군 협업 능력
* 기술 선택 및 의사결정에 대한 근거 제시 능력
* 코드 리뷰 참여 및 문제 조정 능력
5. 테스트 및 코드 품질 관리 (10점)
* 테스트 도구(JUnit, Mockito, PyTest 등) 활용
* Clean Code, SOLID, 리팩터링 경험
* CI/CD 환경에서 테스트/빌드 자동화 경험

**다른 직군 경험과의 차이점 (평가 시 주의사항)**
* Frontend 개발 경험을 Backend 역량으로 평가할 때:
  - UI/UX 구현 경험 ≠ Backend API 설계: Frontend는 사용자 인터페이스 구현을 의미하지만, Backend는 서버 로직, 데이터베이스 설계, API 설계를 의미합니다.
  - 프론트엔드 성능 최적화 ≠ Backend 시스템 확장성: 프론트엔드는 렌더링 최적화, 번들 최적화를 의미하지만, Backend는 대용량 트래픽 처리, 캐싱 전략, 메시지큐 활용을 의미합니다.
  - 브라우저 호환성 대응 ≠ 서버 인프라 운영: 프론트엔드는 크로스 브라우저 대응을 의미하지만, Backend는 서버 운영, 장애 대응, 모니터링을 의미합니다.
* PM/기획 경험을 Backend 역량으로 평가할 때:
  - 프로젝트 관리 경험 ≠ 기술적 아키텍처 설계: PM은 일정 관리, 이해관계자 조율을 의미하지만, Backend는 시스템 아키텍처 설계, 기술 스택 선택, 성능 최적화를 의미합니다.
* 평가 시 다른 직군 경험이 Backend 역량의 일부 요소에만 해당할 수 있음을 고려하고, 보수적으로 평가하세요.
