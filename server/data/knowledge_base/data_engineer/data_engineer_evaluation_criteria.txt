데이터 엔지니어 평가 기준 (총점 100점)
1. 데이터 모델링 및 저장 구조 설계 (30점)
* 데이터 모델링 경험(스타 스키마, 스노우플레이크, 정규화/비정규화 등)
* OLTP / OLAP 구조 이해 및 구축 경험
* 데이터 웨어하우스, 데이터마트, 데이터 레이크 구성 경험
* 대규모 데이터 처리에 적합한 스키마 설계 역량
2. 데이터 파이프라인 및 ETL/ELT 구축 능력 (25점)
* Batch/Streaming 데이터 파이프라인 설계 및 구현 능력
* ETL/ELT 처리 자동화 경험 (Airflow, Spark, Glue, Beam 등)
* 데이터 품질 관리(DQ), 모니터링 및 오류 처리 능력
* 실시간 데이터 처리 및 메시징 시스템 경험(Kafka, Kinesis 등)
3. 데이터 처리 기술 및 성능 최적화 (20점)
* SQL 튜닝, 파티셔닝, 인덱싱 등 고성능 처리 기술
* 분산처리 프레임워크(Spark, Hadoop, Presto, Flink 등) 활용 능력
* 대용량 데이터 성능 개선 및 병목 해결 경험
4. 클라우드 및 플랫폼 이해도 (15점)
* AWS/GCP/Azure 환경에서 Data Platform 구성 경험
* Managed 데이터 서비스 활용(Redshift, BigQuery, Snowflake, Databricks 등)
* 데이터 보안, 권한 관리, GDPR/PII 마스킹 등 컴플라이언스 적용 경험
5. 협업 및 커뮤니케이션 (10점)
* 데이터 분석가/BI/ML 엔지니어와 협업 경험
* 요구사항을 데이터 모델로 해석하고 설계하는 역량
* 기술 문서화 및 파이프라인 운영 문서 작성 능력

**다른 직군 경험과의 차이점 (평가 시 주의사항)**
* Backend 개발 경험을 Data Engineer 역량으로 평가할 때:
  - 애플리케이션 DB 설계 ≠ 데이터 웨어하우스 설계: Backend는 OLTP 데이터베이스 설계를 의미하지만, Data Engineer는 OLAP, 데이터 웨어하우스, 데이터 레이크 설계를 의미합니다.
  - API 개발 경험 ≠ ETL/ELT 파이프라인 구축: Backend는 REST API, 서버 로직 개발을 의미하지만, Data Engineer는 데이터 파이프라인, 배치/스트리밍 처리, 데이터 변환을 의미합니다.
  - 서버 운영 경험 ≠ 데이터 플랫폼 운영: Backend는 애플리케이션 서버 운영을 의미하지만, Data Engineer는 데이터 플랫폼, 데이터 품질 관리, 데이터 거버넌스를 의미합니다.
* ML/AI 엔지니어 경험을 Data Engineer 역량으로 평가할 때:
  - 모델 학습/튜닝 ≠ 데이터 파이프라인 구축: ML/AI는 모델 개발, 학습을 의미하지만, Data Engineer는 데이터 수집, 변환, 저장 파이프라인을 의미합니다.
* 평가 시 다른 직군 경험이 Data Engineer 역량의 일부 요소에만 해당할 수 있음을 고려하고, 보수적으로 평가하세요.
