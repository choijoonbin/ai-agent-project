ML / AI 엔지니어 평가 기준 (총점 100점)
1. Machine Learning / Deep Learning 기술 역량 (30점)
* 회귀/분류/클러스터링 등 ML 알고리즘 이해 및 적용 경험
* 딥러닝 모델(CNN/RNN/Transformer 등) 구조 이해 및 구현 능력
* 모델 학습/튜닝/평가(MLOps Metrics, Hyperparameter tuning 등) 경험
* Feature Engineering 및 데이터 전처리 능력
2. 데이터 처리 및 파이프라인 구축 능력 (20점)
* 대규모 데이터 처리(Spark, Hadoop, Ray 등) 경험
* 데이터 전처리, 증강, 샘플링, 피처 생성 및 파이프라인 구성
* 실험 관리/버전 관리(DVC, MLflow, WandB 등) 경험
3. 모델 배포 및 운영(MLOps) (20점)
* 모델 서빙 기술(FastAPI, Flask, gRPC, Triton, SageMaker 등) 경험
* CI/CD, 모델 재학습 파이프라인 구축 경험
* 모니터링 및 Drift/Degradation 대응 경험
* GPU/CPU 선택 기준 및 성능 최적화 능력
4. 연구/논문 분석 능력 (15점)
* 최신 AI 논문 분석 및 실습 경험
* 모델 아키텍처 개선 또는 커스터마이징 경험
* 벤치마킹 및 실험 설계 능력
5. 협업, 문제 해결 및 커뮤니케이션 (15점)
* 비즈니스 문제를 ML 문제로 정의하는 능력
* 모델 성능 개선과 비용/속도 트레이드오프 설명 능력
* 개발/데이터/PM/사업 조직과 협업 및 기술 설명 능력

**다른 직군 경험과의 차이점 (평가 시 주의사항)**
* Backend 개발 경험을 ML/AI 역량으로 평가할 때:
  - 서버 로직 개발 ≠ 머신러닝 모델 개발: Backend는 비즈니스 로직, API 개발을 의미하지만, ML/AI는 알고리즘 선택, 모델 학습, 하이퍼파라미터 튜닝을 의미합니다.
  - 데이터베이스 쿼리 최적화 ≠ Feature Engineering: Backend는 SQL 쿼리 최적화를 의미하지만, ML/AI는 데이터 전처리, 피처 생성, 피처 선택을 의미합니다.
  - API 서버 운영 ≠ 모델 서빙 및 MLOps: Backend는 REST API 서버 운영을 의미하지만, ML/AI는 모델 서빙, 모델 버전 관리, Drift 감지를 의미합니다.
* Data Engineer 경험을 ML/AI 역량으로 평가할 때:
  - 데이터 파이프라인 구축 ≠ 모델 학습 파이프라인: Data Engineer는 ETL/ELT 파이프라인을 의미하지만, ML/AI는 모델 학습, 평가, 배포 파이프라인을 의미합니다.
  - 데이터 저장/변환 ≠ 모델 실험 관리: Data Engineer는 데이터 웨어하우스 구축을 의미하지만, ML/AI는 실험 추적, 모델 버전 관리, 성능 비교를 의미합니다.
* 평가 시 다른 직군 경험이 ML/AI 역량의 일부 요소에만 해당할 수 있음을 고려하고, 보수적으로 평가하세요.
