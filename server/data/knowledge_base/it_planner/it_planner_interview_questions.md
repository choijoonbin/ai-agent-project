IT 기획자 면접 평가 질문 예시 모음 (별도 정리)
1. 요구사항 수집 및 분석 능력 질문

다양한 이해관계자로부터 요구사항을 수집할 때 어떤 방식(워크숍, 인터뷰, VOC 등)을 활용했는지 설명해주세요.

요구사항 충돌이 발생한 사례가 있다면, 우선순위를 어떤 기준으로 결정했는지 설명해주세요.

요구사항이 변경되거나 범위가 확대되었을 때 어떻게 관리했습니까?

비즈니스 요구를 기술 요구로 변환한 사례를 설명해주세요.

2. 서비스/업무 프로세스 설계 관련 질문

As-Is / To-Be 개선 프로젝트 경험을 상세히 설명해주세요.

설계했던 주요 프로세스 중 가장 복잡한 케이스를 설명하고, 어떻게 단순화했는지 말해주세요.

사용자 여정(User Journey) 또는 Use Case를 작성한 실제 경험을 설명해주세요.

화면 설계서 또는 스토리보드를 작성할 때 어떤 기준을 고려하나요?

3. 문서화 및 산출물 작성 질문

요구사항 정의서 또는 화면 설계서를 작성할 때 포함해야 한다고 생각하는 핵심 요소는 무엇인가요?

본인이 작성한 산출물 중 가장 완성도 높았던 문서를 소개해주세요.

API 명세서를 작성하거나 백엔드 스펙을 정의한 경험이 있나요? 어떤 구조로 정의했습니까?

4. 협업 및 커뮤니케이션 질문

개발팀 또는 디자인팀과 의견 충돌이 발생한 경험과 해결 과정을 설명해주세요.

PM/운영/고객/개발자 등 다양한 역할을 조정해야 했던 경험을 설명해주세요.

전달력이 부족하거나 이해관계자가 기술적 내용을 이해하지 못했을 때 어떻게 설명했나요?

5. 프로젝트 성과 및 개선 경험 질문

성공적으로 끝낸 프로젝트 사례와 구체적인 성과 지표를 말해주세요.

실패한 프로젝트 경험이 있다면 원인과 개선 활동을 설명해주세요.

의사결정 과정에서 데이터 기반 판단을 수행한 경험이 있나요?
